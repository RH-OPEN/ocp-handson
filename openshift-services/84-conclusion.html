{lab_name} をお楽しみいただけたでしょうか。そして、OpenShift Virtualization と Ansible Automation Platform、および Red Hat のこれら 2 つのツールが、仮想化管理者として、仮想マシン資産を日常的に管理する方法、および必要なパフォーマンス分析とトラブルシューティング手順を実行する方法について、さらに学んでいただけたことを願っています。

主なポイント

  • OpenShift Virtualization はソリューション指向であり、使い始めるのが簡単な、成熟し、安定し、スケーラブルなソリューションを提供します。

  • Ansible Automation Platform は、仮想マシン環境の多くの機能を大規模に自動化および管理する機能を提供するエンタープライズツールです。

Red Hat Learning Services: DO317 Red Hat OpenShift Virtualization を使用した仮想マシンの管理

  • OpenShift クラスター上で VM を作成、アクセス、および管理するために必要なスキル。

  • コンテナーに対してもこれらのリソースの使用法とアクセスを制御するのと同じ Kubernetes 機能を使用して、VM からの CPU、メモリー、ストレージ、およびネットワークリソースの使用法とアクセスを制御するために必要なスキル。

  • 標準の Kubernetes 機能と OpenShift Virtualization からの拡張機能を使用した VM の高可用性 (HA) を管理するためのサンプルアーキテクチャ。

  • ストレージやデータベースなど、OpenShift クラスター外のデータセンターサービスに OpenShift 上の VM を接続するための戦略。

  • Migration Toolkit for Virtualization オペレーターを使用して、互換性のあるハイパーバイザーから OpenShift Virtualization に VM を移行するための戦略。 このリンク をクリックして確認してください。

Red Hat Learning Services: AU374 Red Hat Ansible Automation Platform を使用した高度な自動化の開発

  • 自動化コンテンツナビゲーターの機能を活用して Ansible Playbook を開発する。

  • Ansible を使用した効果的かつ効率的な自動化のための推奨プラクティスを適用する。

  • Red Hat Ansible Automation Platform の高度な機能を使用して、フィルターやプラグインなどのデータを操作する。

  • ローリングアップデートとして自動化操作を実行する。

  • 自動化コードを実行するために必要な依存関係をバンドルおよび配布するための自動化実行環境を作成する。 このリンク をクリックして確認してください。
